# MCP Memory Plugin Scorecards

This page indexes the latest scorecard for every MCP memory plugin. Each plugin's
scorecard follows the §7.4 template in
[../proposals/mcp_memory_plugin_manager.md#74-the-plugin-scorecard-template](../proposals/mcp_memory_plugin_manager.md#74-the-plugin-scorecard-template)
and is regenerated by `make eval-plugin PLUGIN=<name>` on every PR that touches the
plugin. Frozen baselines live under [./baseline_v1/](./baseline_v1/) and are never
overwritten without a `CHANGELOG.md` entry.

| Plugin      | Latest run     | Scorecard                                                                              | Status                     |
| ----------- | -------------- | -------------------------------------------------------------------------------------- | -------------------------- |
| `rag`       | Phase-1 ship   | [baseline_v1/rag_plugin_scorecard.md](baseline_v1/rag_plugin_scorecard.md)             | frozen baseline_v1         |
| `pageindex` | Phase-2 wave-B | [baseline_v1/pageindex_plugin_scorecard.md](baseline_v1/pageindex_plugin_scorecard.md) | phase 2 — datasets pending |

The pageindex row links to a placeholder. The nightly CI workflow will regenerate
`docs/eval/runs/<sha>/pageindex_plugin_scorecard.md` once the §7.3 golden datasets
land via LFS; until then every cell is `n/a`.

This index updates as new plugins ship. Future plugins follow the same pattern.

For the harness runbook (running locally, capturing a new baseline, interpreting CI
output) see [./README.md](./README.md).
